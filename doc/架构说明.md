# Nuist CampusWall 架构说明（V2.1）

## 1. 分层架构
1. `controller`：接口层
2. `service`：业务层
3. `mapper`：数据访问层（MyBatis-Plus）
4. `domain`：实体层
5. `dto`：请求/响应对象
6. `common`：统一返回、异常、错误码
7. `security`：JWT 与用户上下文
8. `config`：MVC 与 MP 配置

## 2. 核心流程
### 2.1 点赞流程（新增）
`LikeController.like`
-> `LikeServiceImpl.like`
-> 登录校验（`UserContext`）
-> 校验目标存在（`post`/`comment`）
-> 查重（`like` 表）
-> 插入点赞记录
-> 更新目标 `like_count + 1`

### 2.2 取消点赞流程（新增）
`LikeController.unlike`
-> `LikeServiceImpl.unlike`
-> 登录校验
-> 校验点赞记录存在
-> 删除点赞记录
-> 更新目标 `like_count - 1`（下限 0）

## 3. 鉴权与异常
1. JWT 拦截 `/api/**`，放行登录注册。
2. 业务异常统一 `BusinessException`。
3. 全局异常统一 `GlobalExceptionHandler`。
4. 统一返回 `Result<T>`。

## 4. 一致性策略
1. `like` 表唯一键防重复点赞。
2. 评论回复字段保持成对一致。
3. 枚举通过 `@EnumValue` 持久化。

## 5. 当前技术风险与优化方向
1. 点赞计数当前为“查后改”，并发下可能有竞争。
2. 后续建议改为 SQL 原子增减（`like_count = like_count + 1/-1`）。
