# Nuist CampusWall 架构说明（V2.0）

## 1. 分层架构
项目采用典型 Spring Boot 分层：
1. `controller`：接口层，请求接收、参数绑定、返回包装。
2. `service`：业务层，业务校验与流程编排。
3. `mapper`：数据访问层，MyBatis-Plus 持久化。
4. `domain`：实体层，对应数据库表。
5. `dto`：接口输入输出对象。
6. `common`：统一返回、错误码、异常体系。
7. `security`：JWT 工具、鉴权拦截、用户上下文。
8. `config`：MVC 拦截器注册、MP 分页配置、启动检查。

## 2. 核心流程
### 2.1 注册流程
`AccountController.register`
-> `AccountServiceImpl.register`
-> `UserMapper.selectCount`（用户名查重）
-> `UserMapper.selectCount`（邮箱查重）
-> BCrypt 加密密码
-> `UserMapper.insert`

### 2.2 登录流程
`AccountController.login`
-> `AccountServiceImpl.login`
-> `UserMapper.selectOne`（按用户名）
-> 用户状态校验
-> BCrypt 密码校验
-> `JwtUtil.generateToken`
-> 返回 `LoginRespVO(token + userInfo)`

### 2.3 鉴权流程
`WebMvcConfig` 注册 `JwtAuthInterceptor`
-> 请求进入拦截器
-> 校验 `Authorization: Bearer <token>`
-> `JwtUtil.parseToken`
-> 写入 `UserContext`
-> 业务层读取当前用户ID

### 2.4 发帖流程
`PostController.create`
-> DTO 校验
-> `PostServiceImpl.create`
-> 从 `UserContext` 取 userId
-> 组装 `Post` 并写库

### 2.5 评论流程
`CommentController.create`
-> DTO 校验
-> `CommentServiceImpl.create`
-> 从 `UserContext` 取 userId
-> 校验 `postId` 对应帖子存在
-> 组装 `Comment` 并写库

## 3. 安全与异常
1. 密码安全：`spring-security-crypto` + BCrypt。
2. 接口鉴权：JWT 拦截器保护 `/api/**`。
3. 统一异常：`BusinessException` + `GlobalExceptionHandler`。
4. 参数异常：`MethodArgumentNotValidException` 统一映射 `422`。

## 4. 数据一致性策略
1. 用户名/邮箱查重放在 Service 层。
2. 数据库唯一索引兜底（防并发重复）。
3. 评论回复对（`reply_to_comment_id`、`reply_to_user_id`）使用 CHECK 保证一致性。
4. 点赞行为表唯一键防重复点赞。

## 5. 技术关键点
1. 使用 `mybatis-plus-spring-boot4-starter`。
2. 启用 `mybatis-plus-jsqlparser`，并在 `MybatisPlusConfig` 配置分页拦截器。
3. 枚举使用 `@EnumValue` 持久化为数据库数值。
4. 返回体统一为 `Result<T>`。

## 6. 当前包结构注意项
当前 DTO 中账号相关类在路径：
`com.nuist_campuswall.dto.comment.account`

该路径可运行，但语义上建议后续重构为：
`com.nuist_campuswall.dto.account`

本轮不改代码，仅在文档中明确现状。
