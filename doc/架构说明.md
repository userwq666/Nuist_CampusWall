# Nuist CampusWall 架构说明（V1）

## 1. 分层架构
项目采用典型后端分层：
1. `controller`：接收请求、参数绑定、返回统一结果。
2. `service`：业务编排与业务校验（如查重、加密、状态判断）。
3. `mapper`：数据访问层（MyBatis-Plus）。
4. `domain`：领域对象（与数据库表结构对应）。
5. `dto`：接口入参/出参对象（与 domain 解耦）。
6. `common`：通用组件（统一返回、业务异常、全局异常处理）。

## 2. 当前核心调用链（注册）
`AccountController.register`  
-> `AccountService.register`  
-> `UserMapper.selectCount`（用户名查重）  
-> `UserMapper.selectCount`（邮箱查重）  
-> BCrypt 密码加密  
-> `UserMapper.insert` 入库  
-> 返回 `Result.success(...)`

## 3. 数据一致性策略
1. 业务查重：Service 层先查后写。
2. 数据库兜底：用户名、邮箱唯一索引保证最终一致性。
3. 评论回复一致性：`reply_to_comment_id` 与 `reply_to_user_id` 同空或同非空。
4. 点赞防重：`like(user_id, target_type, target_id)` 唯一键。

## 4. 异常与返回规范
1. 业务异常：抛出 `BusinessException(code, message)`。
2. 全局异常：`GlobalExceptionHandler` 统一拦截并返回 `Result.fail(...)`。
3. 返回结构：`code/message/data`。
