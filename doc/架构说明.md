# Nuist CampusWall 架构说明（V2.5）

## 1. 分层架构
1. `controller`：参数绑定、响应输出。
2. `service`：业务编排与权限校验。
3. `mapper`：MyBatis-Plus 数据访问。
4. `domain`：实体映射。
5. `dto/vo`：输入输出对象。
6. `common`：统一返回、异常、错误码。
7. `security`：JWT、拦截器、用户上下文。

## 2. admin 分包
1. `service.admin.auth`：管理员权限校验。
2. `service.admin.user`：管理员用户业务（当前为分页）。

## 3. 帖子分流设计（新增）
1. `page`：查询 `status=ENABLE` 且 `user_id != app.admin-user-id`。
2. `noticePage`：查询 `status=ENABLE` 且 `user_id = app.admin-user-id`。
3. 公告管理采用配置项，不写死管理员ID。

## 4. 下一阶段
1. `service.admin.user` 增加启用/禁用。
2. 增加 `service.admin.post` 与 `service.admin.comment`。
