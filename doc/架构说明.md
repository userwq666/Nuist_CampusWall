# Nuist CampusWall 架构说明（V2.3）

## 1. 分层架构
1. `controller`：接收请求、参数绑定、返回统一响应。
2. `service`：业务编排、权限判断、异常抛出。
3. `mapper`：MyBatis-Plus 数据访问。
4. `domain`：实体映射数据库表。
5. `dto/vo`：请求参数与响应对象。
6. `common`：`Result`、`BusinessException`、`ErrorCode`。
7. `security`：JWT 工具、拦截器、用户上下文。
8. `config`：MVC 拦截器注册、MyBatis-Plus 分页插件等。

## 2. 鉴权机制
1. 登录成功签发 JWT，写入用户关键信息（`userId`、`username`、`role`、`status`）。
2. 拦截器校验 token，合法后写入 `UserContext`。
3. 业务层从 `UserContext` 获取当前用户执行权限判断。

## 3. 模块调用链路
### 3.1 注册
`AccountController.register`
-> `AccountService.register`
-> 用户名/邮箱查重
-> BCrypt 加密
-> `UserMapper.insert`

### 3.2 登录
`AccountController.login`
-> `AccountService.login`
-> `UserMapper` 查询用户
-> BCrypt 匹配
-> `JwtUtil.generateToken`
-> 返回 `token + userInfo`

### 3.3 点赞
`LikeController.do/undo`
-> `LikeService`
-> 目标资源校验（post/comment）
-> 点赞记录增删
-> 目标 `like_count` 更新

## 4. 数据一致性与约束
1. 评论回复字段成对约束（`ck_comment_reply_pair`）。
2. 点赞唯一键防重复点赞。
3. 公开列表统一按 `status=ENABLE` 过滤。
4. 软删除避免物理删除带来的追溯困难。

## 5. 下一阶段架构扩展
1. 增加管理员鉴权层（role=ADMIN）。
2. 新增 `admin` 控制器与管理服务。
3. 逐步引入更细粒度权限点。
