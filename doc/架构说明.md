# Nuist CampusWall 架构说明（V3.4）

## 1. 分层结构
1. `controller`：接收参数、返回结果。
2. `service`：业务编排、权限校验、状态流转。
3. `mapper`：MyBatis-Plus 数据访问。
4. `domain`：数据库实体映射。
5. `dto`：接口输入输出模型。
6. `common`：统一返回、异常、错误码、映射器。
7. `security`：JWT 与用户上下文。
8. `config`：MVC/分页/初始化/定时任务配置。

## 2. 鉴权链路
1. `JwtAuthInterceptor` 拦截 `/api/**`。
2. 放行注册与登录。
3. 解析 token 后写入 `UserContext`。
4. 管理端通过 `AdminAuthService.adminCheck()` 做角色校验。

## 3. 异常与响应链路
1. 业务层抛出 `BusinessException(code, message)`。
2. `GlobalExceptionHandler` 统一捕获异常。
3. `ErrorCodeToHttpStatus` 负责业务码映射 HTTP 状态。
4. 输出统一结构：`Result{code,message,data}`。

## 4. 文件子系统
1. 上传目录：`temp/yyyy/MM/dd`。
2. 绑定迁移：`post/{id}`、`comment/{id}`、`avatar/{id}`。
3. 解绑回写 `TEMP`。
4. 定时清理超时 `TEMP` 为 `DELETED` 并删除物理文件。

## 5. 架构结论
1. 当前架构满足课程项目可交付与可扩展要求。
2. 后续可平滑扩展消息通知、状态流转、审计日志等能力。
