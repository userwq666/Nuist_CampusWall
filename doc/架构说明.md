# Nuist CampusWall 架构说明（V3.1）

## 1. 总体分层
1. `controller`：参数接收、调用业务、返回统一响应。
2. `service`：业务编排、权限校验、状态流转。
3. `mapper`：MyBatis-Plus 数据访问。
4. `domain`：数据库实体映射。
5. `dto`：接口输入输出对象。
6. `common`：统一返回、异常、错误码。
7. `security`：JWT 工具、鉴权拦截器、用户上下文。
8. `config`：MyBatis-Plus、WebMVC、初始化检查、定时任务。

## 2. 鉴权与权限控制
1. `JwtAuthInterceptor` 拦截 `/api/**`，放行注册/登录。
2. `UserContext(ThreadLocal)` 保存当前用户 ID。
3. 管理员操作统一走 `AdminAuthService.adminCheck()`。
4. 管理员判定依据为 `Role.ADMIN`。

## 3. 模块设计
### 3.1 用户端模块
1. account：注册、登录、我的信息、修改我的信息。
2. file：上传、绑定、解绑、删除与生命周期管理。
3. post：创建、公开分页、公告分页、我的分页、详情、修改、删除。
4. comment：创建、公开分页、我的分页、删除。
5. like：点赞、取消点赞。

### 3.2 管理端模块（按能力分包）
1. `service.admin.auth`：管理员鉴权。
2. `service.admin.user`：用户分页与启用/禁用。
3. `service.admin.post`：帖子分页、详情、启用/禁用。
4. `service.admin.comment`：评论分页、详情、启用/禁用。

## 4. 文件子系统设计
1. 上传入口：`POST /api/file/upload`。
2. 上传目录策略：先落 `temp/yyyy/MM/dd`。
3. 绑定迁移策略：
   - 帖子：`post/{bizId}/...`
   - 评论：`comment/{bizId}/...`
   - 头像：`avatar/{bizId}/...`
4. 状态机：`TEMP(0) -> BOUND(1) -> TEMP(解绑) -> DELETED(2)`。
5. 定时清理：`FileCleanTask` 按 `updateTime` 清理超时 `TEMP` 文件。

## 5. 数据层关键设计
1. 状态字段统一软删除治理：`UserStatus/PostStatus/CommentStatus`。
2. 点赞表联合唯一索引防重复点赞。
3. 文件表 `file_asset` 通过 `file_type/biz_type/status` 约束保障一致性。
4. 公告分流通过 `app.admin-user-id` + `post.user_id` 条件实现。

## 6. 异常与可维护性
1. 业务异常统一抛 `BusinessException(code, message)`。
2. `GlobalExceptionHandler` 统一处理业务异常、参数异常、兜底异常。
3. 错误码集中维护于 `ErrorCode`，避免魔法数字分散。

## 7. 结论
1. 当前架构已满足课程项目交付与答辩需求。
2. 后续扩展（消息通知、状态流转、前端联调）可在现有结构平滑增加。
