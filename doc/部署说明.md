# Nuist CampusWall 部署说明（开发环境）

## 1. 环境要求
1. JDK 17
2. Maven 3.9+
3. MySQL 8
4. IntelliJ IDEA（推荐）

## 2. 数据库准备
1. 确保 MySQL 可连接。
2. 检查 `application.properties` 中数据库配置：
   - `spring.datasource.url`
   - `spring.datasource.username`
   - `spring.datasource.password`
3. 项目启动时会按配置自动执行：
   - `src/main/resources/sql/db_init.sql`
   - `src/main/resources/sql/db_seed.sql`

## 3. 启动项目
在项目根目录执行：

```bash
mvn spring-boot:run
```

默认端口：
1. `8080`（可在 `application.properties` 的 `server.port` 修改）

## 4. 快速验证
1. 打开 [src/test/http/account.http](../src/test/http/account.http)
2. 打开 [src/test/http/post.http](../src/test/http/post.http)
3. 运行注册、登录、`/api/account/me` 与 `post/create` 测试用例
3. 预期结果：
   - 成功注册返回 `code=0`
   - 重复用户名返回业务错误码（`401`）
   - 重复邮箱返回业务错误码（`402`）
   - 登录成功返回 `code=0` 且 `data` 包含 `token + userInfo`
   - 登录失败（密码错误）返回业务错误码（`405`）
   - 不带token访问 `/api/account/me` 返回业务错误码（`406`）
   - 带token发帖成功返回 `code=0`
   - 不带token发帖返回业务错误码（`406`）
   - 帖子参数不合法返回业务错误码（`422`）

## 5. 常见问题
1. 端口占用
   - 报错：`Port 8080 was already in use`
   - 处理：修改 `server.port` 或关闭占用进程
2. 数据库连接失败
   - 检查 MySQL 是否启动
   - 检查用户名、密码、数据库地址
3. SQL 初始化失败
   - 检查 `db_init.sql` 语法
   - 检查表名与关键字冲突情况
4. 接口返回 500
   - 查看控制台日志堆栈
   - 检查全局异常处理返回信息

## 6. 当前部署范围说明
本说明用于本地开发与课程演示环境，不包含生产环境部署（Nginx、容器化、CI/CD）。
