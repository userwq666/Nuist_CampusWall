# Nuist CampusWall 部署说明（开发环境，V3.4）

最后同步：2026-04-03

## 1. 环境要求
1. JDK 21
2. Maven 3.9+
3. MySQL 8.x
4. Node.js 20+
5. IntelliJ IDEA / WebStorm（UTF-8 编码）

## 2. Windows 终端编码
在 PowerShell 执行：
```powershell
chcp 65001
[Console]::InputEncoding  = [System.Text.Encoding]::UTF8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$OutputEncoding           = [System.Text.Encoding]::UTF8
```

## 3. 后端配置项（application.properties）
### 3.1 数据库
1. `spring.datasource.url`
2. `spring.datasource.username`
3. `spring.datasource.password`
4. `spring.datasource.driver-class-name`

### 3.2 初始化脚本
1. `spring.sql.init.mode=always`
2. `spring.sql.init.schema-locations=classpath:sql/db_init.sql`
3. `spring.sql.init.data-locations=classpath:sql/db_seed.sql`

### 3.3 JWT
1. `jwt.secret`
2. `jwt.expire-seconds`
3. `jwt.issuer`

### 3.4 文件模块
1. `app.file.local-root`（建议英文目录，如 `D:/campuswall_upload`）
2. `app.file.url-prefix`
3. `app.file.max-image-size`
4. `app.file.clean-temp-cron`

### 3.5 上传限制
1. `spring.servlet.multipart.max-file-size`
2. `spring.servlet.multipart.max-request-size`

建议：multipart 上限应大于业务上限（例如 10MB > 5MB）。

## 4. 启动方式
### 4.1 后端
```bash
mvn spring-boot:run
```

或

```bash
mvn clean package
java -jar target/Nuist_CampusWall-0.0.1-SNAPSHOT.jar
```

### 4.2 前端
在 `frontend` 目录：
```bash
npm install
npm run dev
```

## 5. 初始化策略
1. `db_init.sql`：全量建表脚本（课程项目默认方案）。
2. `db_seed.sql`：管理员种子数据（不存在才插入）。
3. 当前不使用 Flyway/Liquibase 增量迁移。

## 6. 回归建议顺序
1. `src/test/http/account.http`
2. `src/test/http/file.http`
3. `src/test/http/post.http`
4. `src/test/http/comment.http`
5. `src/test/http/like.http`
6. `src/test/http/admin.http`

断言规则：`HTTP 状态码 + 业务 code`。

## 7. HTTP Client 环境变量
文件：`src/test/http/http-client.env.json`
1. `host`：如 `http://localhost:8080`
2. `token`：普通用户登录后 token
3. `adminToken`：管理员登录后 token

## 8. 常见问题排查
1. 启动不建表：检查 `spring.sql.init.*` 与数据库连接。
2. 上传 500/413：检查业务大小限制与 multipart 上限。
3. 上传成功但找不到文件：核对 `app.file.local-root`、`object_key`、磁盘路径。
4. 管理接口 403/412：确认当前 token 用户角色是否为 `ADMIN`。
5. 控制台中文乱码：确认终端与 IDE 编码为 UTF-8。
6. 前端接口失败：确认 `request.js` 的 `baseURL` 与后端端口一致。
