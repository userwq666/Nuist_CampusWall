# Nuist CampusWall 部署说明（开发环境，V3.1）

## 1. 环境要求
1. JDK 21
2. Maven 3.9+
3. MySQL 8.x
4. IntelliJ IDEA（项目编码统一 UTF-8）

## 2. 编码与终端（Windows）
项目默认 UTF-8，PowerShell 建议先执行：
```powershell
chcp 65001
[Console]::InputEncoding  = [System.Text.Encoding]::UTF8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$OutputEncoding           = [System.Text.Encoding]::UTF8
```

## 3. 配置准备
编辑 `src/main/resources/application.properties`：
1. `spring.datasource.url`
2. `spring.datasource.username`
3. `spring.datasource.password`
4. `jwt.secret`
5. `jwt.expire-seconds`
6. `jwt.issuer`
7. `app.admin-user-id`
8. `app.file.max-image-size`
9. `spring.servlet.multipart.max-file-size`
10. `spring.servlet.multipart.max-request-size`
11. `app.file.local-root`
12. `app.file.url-prefix`
13. `app.file.clean-temp-cron`

建议：
1. `app.file.local-root` 使用英文路径（如 `D:/campuswall_upload`）。
2. multipart 上限应大于业务限制（例如 10MB > 5MB）。

## 4. 数据库初始化策略
1. 启动时自动执行：
   - `src/main/resources/sql/db_init.sql`
   - `src/main/resources/sql/db_seed.sql`
2. 管理员种子账号由 `db_seed.sql` 写入（仅不存在时插入）。
3. 当前采用全量初始化脚本方案，适用于课程项目演示环境。

## 5. 启动方式
```bash
mvn spring-boot:run
```

或：
```bash
mvn clean package
java -jar target/Nuist_CampusWall-0.0.1-SNAPSHOT.jar
```

## 6. 本地回归建议顺序
1. `src/test/http/account.http`
2. `src/test/http/file.http`
3. `src/test/http/post.http`
4. `src/test/http/comment.http`
5. `src/test/http/like.http`
6. `src/test/http/admin.http`

## 7. HTTP 环境变量
文件：`src/test/http/http-client.env.json`
1. `host`：例如 `http://localhost:8080`
2. `token`：普通用户登录后 token
3. `adminToken`：管理员登录后 token

## 8. 常见问题排查
1. 启动不建表：检查数据库连接与 `spring.sql.init.*` 配置。
2. 上传文件返回 500：优先检查 multipart 上限与业务文件上限关系。
3. 文件目录看不到文件：核对 `app.file.local-root` 与数据库 `object_key`。
4. 管理员接口返回 412：确认 token 对应角色为 `ADMIN`。
5. 控制台乱码：确认终端与 IDEA 编码均为 UTF-8。
