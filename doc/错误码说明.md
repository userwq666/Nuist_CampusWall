# Nuist CampusWall 错误码说明（V1.3）

## 1. 设计目标
1. 让前端可稳定识别失败原因。
2. 统一后端错误表达，减少硬编码字符串判断。
3. 便于后续扩展登录、鉴权与业务模块。

## 2. 当前错误码
| 错误码 | 含义     | 说明        |
|------|--------|-----------|
| 0    | 成功     | 请求处理成功    |
| 401  | 用户名已存在 | 注册时用户名重复  |
| 402  | 邮箱已存在  | 注册时教育邮箱重复 |
| 403  | 用户名不存在 | 登录时账号不存在  |
| 404  | 用户已被禁用 | 登录时状态校验拦截 |
| 405  | 密码错误   | 登录时密码不匹配  |
| 406  | 未登录或token缺失 | 鉴权拦截时未携带token |
| 407  | token无效或已过期 | 鉴权拦截时解析或验签失败 |
| 408  | 用户不存在 | `/api/account/me` 查询用户为空 |
| 422  | 参数校验失败 | DTO 注解校验失败（如标题为空、内容超长） |
| 500  | 服务器异常  | 未分类异常兜底   |

## 3. 使用规范
1. 成功统一返回：`Result.success(data)`，`code=0`。
2. 业务失败统一抛出：`BusinessException(code, message)`。
3. 控制器不直接拼接错误 JSON，统一交给全局异常处理。
4. 新增错误码时，同步更新本文件与 API 文档。

## 4. 维护约定
1. 错误码统一在 `ErrorCode.java` 维护，文档与代码保持一致。
2. 新增错误码时同步更新：`ErrorCode.java`、`API文档.md`、本文件。
