# Nuist CampusWall 错误码说明（V1）

## 1. 设计目标
1. 让前端可稳定识别失败原因。
2. 统一后端错误表达，减少硬编码字符串判断。
3. 便于后续扩展登录、鉴权与业务模块。

## 2. 当前错误码
| 错误码  | 含义 | 说明 |
|------|---|---|
| 0    | 成功 | 请求处理成功 |
| 4001 | 用户名已存在 | 注册时用户名重复 |
| 4002 | 邮箱已存在 | 注册时教育邮箱重复 |
| 5000 | 服务器异常 | 未分类异常兜底 |

## 3. 使用规范
1. 成功统一返回：`Result.success(data)`，`code=0`。
2. 业务失败统一抛出：`BusinessException(code, message)`。
3. 控制器不直接拼接错误 JSON，统一交给全局异常处理。
4. 新增错误码时，同步更新本文件与 API 文档。

## 4. 后续预留（建议）
1. `401x`：认证类错误（未登录、令牌过期）。
2. `403x`：授权类错误（权限不足）。
3. `404x`：资源不存在（帖子/评论不存在）。
4. `422x`：参数校验错误。
