# Nuist CampusWall 错误码说明（V2.1）

## 1. 设计原则
1. 成功统一 `code=0`
2. 业务失败抛 `BusinessException`
3. 错误码统一定义于 `ErrorCode.java`
4. 全局异常统一输出 `Result.fail(...)`

## 2. 错误码明细
| code | 常量名 | 含义 |
|---|---|---|
| 0 | - | 成功 |
| 401 | USERNAME_EXISTS | 用户名已存在 |
| 402 | EMAIL_EXISTS | 邮箱已存在 |
| 403 | USERNAME_NOT_FOUND | 用户名不存在 |
| 404 | USER_DISABLED | 用户已被禁用 |
| 405 | PASSWORD_INCORRECT | 密码错误 |
| 406 | UNAUTHORIZED | 未登录或 token 缺失 |
| 407 | TOKEN_INVALID | token 无效/过期 |
| 408 | USER_NOT_FOUND | 用户不存在 |
| 409 | POST_NOT_FOUND | 帖子不存在 |
| 410 | COMMENT_NOT_FOUND | 评论不存在 |
| 411 | LIKE_EXIST | 点赞已存在（重复点赞） |
| 422 | PARAMETER_ERROR | 参数校验失败/业务参数错误 |
| 500 | SERVER_ERROR | 服务器异常 |

## 3. 使用约定
1. Controller 不硬编码错误码。
2. 统一从 `ErrorCode` 常量读取。
3. 新增错误码必须同步更新 `API文档` 与本文件。
