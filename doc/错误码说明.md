# Nuist CampusWall 错误码说明（V3.3）

## 1. 规则
1. 返回体固定三段式：`{ code, message, data }`。
2. `code=0` 表示业务成功，`code!=0` 表示业务失败。
3. HTTP 状态码用于表示失败类型（401/403/404/409/413/415/422/500 等）。

## 2. 常见业务码
| code | 常量 | 说明 |
|---|---|---|
| 0 | - | 成功 |
| 401 | USERNAME_EXISTS | 用户名已存在 |
| 402 | EMAIL_EXISTS | 邮箱已存在 |
| 403 | USERNAME_NOT_FOUND | 用户名不存在 |
| 404 | USER_DISABLED | 用户被禁用 |
| 405 | PASSWORD_INCORRECT | 密码错误 |
| 406 | UNAUTHORIZED | 未登录或 token 缺失 |
| 407 | TOKEN_INVALID | token 无效或过期 |
| 408 | USER_NOT_FOUND | token 对应用户不存在 |
| 409 | POST_NOT_FOUND | 帖子不存在 |
| 410 | COMMENT_NOT_FOUND | 评论不存在 |
| 411 | LIKE_EXIST | 重复点赞 |
| 412 | NO_PERMISSION | 无权限 |
| 413 | POST_STATUS_ERROR | 帖子状态错误 |
| 414 | FILE_NOT_FOUND | 文件不存在 |
| 415 | FILE_TYPE_ERROR | 文件类型错误 |
| 416 | FILE_SIZE_ERROR | 文件大小错误 |
| 417 | FILE_STATUS_ERROR | 文件状态错误 |
| 418 | FILE_UPLOAD_ERROR | 文件上传失败 |
| 419 | FILE_DELETE_ERROR | 文件删除失败 |
| 422 | PARAMETER_ERROR | 参数错误 |
| 500 | SERVER_ERROR | 服务器异常 |

## 3. HTTP 与业务码关系
1. `HTTP 200` 常见：`code=0`。
2. `HTTP 401` 常见：`405/406/407`。
3. `HTTP 403` 常见：`412`。
4. `HTTP 404` 常见：`408/409/410/414`。
5. `HTTP 409` 常见：`401/402/411/413/417`。
6. `HTTP 413` 常见：`416`。
7. `HTTP 415` 常见：`415`。
8. `HTTP 422` 常见：`422`。
9. `HTTP 500` 常见：`418/419/500`。

## 4. 真源位置
1. `src/main/java/com/nuist_campuswall/common/ErrorCode.java`：业务码真源。
2. `src/main/java/com/nuist_campuswall/common/ErrorCodeToHttpStatus.java`：业务码到 HTTP 的映射真源。
3. `src/main/java/com/nuist_campuswall/common/GlobalExceptionHandler.java`：异常统一出口。
