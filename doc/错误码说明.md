# Nuist CampusWall 错误码说明（V2.0）

## 1. 设计原则
1. 成功统一 `code=0`
2. 业务失败统一抛 `BusinessException`
3. 错误码统一定义在 `ErrorCode.java`
4. 全局异常统一由 `GlobalExceptionHandler` 返回 `Result.fail(...)`

## 2. 错误码明细
| code | 常量名 | 含义 | 典型场景 |
|---|---|---|---|
| 0 | - | 成功 | 请求正常处理 |
| 401 | USERNAME_EXISTS | 用户名已存在 | 注册时用户名重复 |
| 402 | EMAIL_EXISTS | 邮箱已存在 | 注册时教育邮箱重复 |
| 403 | USERNAME_NOT_FOUND | 用户名不存在 | 登录时账号不存在 |
| 404 | USER_DISABLED | 用户已被禁用 | 登录状态校验失败 |
| 405 | PASSWORD_INCORRECT | 密码错误 | 登录密码不匹配 |
| 406 | UNAUTHORIZED | 未登录或 token 缺失 | 未携带 `Authorization` |
| 407 | TOKEN_INVALID | token 无效/过期 | token 解析失败 |
| 408 | USER_NOT_FOUND | 用户不存在 | `/api/account/me` 按 ID 查无用户 |
| 409 | POST_NOT_FOUND | 帖子不存在 | 评论目标帖子不存在或帖详情不存在 |
| 422 | PARAMETER_ERROR | 参数校验失败 | DTO 注解校验失败 |
| 500 | SERVER_ERROR | 服务器异常 | 未分类异常兜底 |

## 3. 与 HTTP 状态码关系
当前项目使用“业务码 + 统一 HTTP 200”模式：
1. HTTP 状态多为 `200`
2. 真实业务结果看 `code`

后续可选优化：
1. 保持业务码不变
2. 增加 HTTP 状态映射（如 `406->401`，`422->400`）

## 4. 维护约定
1. 新增错误码必须同步更新：
   - `src/main/java/com/nuist_campuswall/common/ErrorCode.java`
   - `doc/错误码说明.md`
   - `doc/API文档.md`
2. 禁止在 Controller 中硬编码错误码数字。
