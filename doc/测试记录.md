# Nuist CampusWall 测试记录（V1.1）

## 1. 测试范围
本轮覆盖账户模块注册与登录接口：
1. `POST /api/account/register`
2. `POST /api/account/login`

## 2. 测试环境
1. 本地开发环境（Windows）
2. Spring Boot 本地运行（默认 `8080`）
3. MySQL 本地实例
4. 测试工具：IntelliJ HTTP Client（`src/test/http/account.http`）

## 3. 用例记录

### 用例1：注册成功
请求：
```json
{
  "username": "test1001",
  "password": "123456",
  "nickname": "测试用户1001",
  "educationEmail": "test1001@nuist.edu.cn"
}
```

预期：
1. 返回 `code=0`
2. 返回成功消息
3. 数据库新增用户记录

结果：
1. 通过

---

### 用例 2：用户名重复
请求：
```json
{
  "username": "test1001",
  "password": "123456",
  "nickname": "测试用户 1002",
  "educationEmail": "test1002@nuist.edu.cn"
}
```

预期：
1. 返回业务错误
2. 错误码为 `401`
3. 错误消息提示用户名已存在

结果：
1. 通过

---

### 用例 3：邮箱重复
请求：
```json
{
  "username": "test1003",
  "password": "123456",
  "nickname": "测试用户 1003",
  "educationEmail": "test1001@nuist.edu.cn"
}
```

预期：
1. 返回业务错误
2. 错误码为 `402`
3. 错误消息提示邮箱已存在

结果：
1. 通过

---

### 用例 4：登录成功
请求：
```json
{
  "username": "test1",
  "password": "123456"
}
```

预期：
1. 返回 `code=0`
2. `data` 为 `LoginVO` 对象（包含 id/username/nickname/role/status）
3. 不返回密码字段

结果：
1. 通过

---

### 用例 5：登录失败（密码错误）
请求：
```json
{
  "username": "test1",
  "password": "1234567"
}
```

预期：
1. 返回业务错误
2. 错误码为 `405`
3. 错误消息提示密码错误

结果：
1. 通过

## 4. 数据验证
1. 用户密码为 BCrypt 加密串，不是明文保存。
2. 用户名与邮箱重复提交会被拦截，不会写入重复数据。
3. 登录成功仅返回用户基础信息，不返回密码。

## 5. 结论
注册与登录主链路已可用，具备继续开发 JWT 鉴权与 `/api/account/me` 的基础。
