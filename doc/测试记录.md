## 3. 断言标准
1. 成功：`HTTP 200 + code=0`。
2. 失败：`HTTP 4xx/5xx + code!=0`。
3. 文案：优先使用返回 `message`。

## 4. 关键失败场景
1. 未登录访问受保护接口（401 + 406）。
2. 普通用户访问管理接口（403 + 412）。
3. 目标资源不存在（404 + 409/410/414）。
4. 重复操作冲突（409 + 411/417）。
5. 参数非法（422 + 422）。
6. 上传超限（413 + 416）。

## 5. 结论
1. 当前后端主链路可稳定复现成功与失败场景。
2. 测试口径与文档口径已统一。
3. 可进入前端联调阶段。
