## 3. 断言规则（已统一）
1. 成功场景：`HTTP 200 + code=0`。
2. 失败场景：`HTTP 4xx/5xx + code!=0`。
3. 页面提示信息优先使用 `message`。

## 4. 本轮同步（2026-04-03）
1. 响应协议回切为三段式：`{ code, message, data }`。
2. 六个 HTTP 用例文件均补充业务码预期说明。
3. 前端 `request.js` 已适配三段式响应。

## 5. 常见状态码与业务码
1. `401`：`405/406/407`。

4. `409`：`401/402/411/413/417`。
