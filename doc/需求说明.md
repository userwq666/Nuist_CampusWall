# Nuist CampusWall 需求说明（V3.4）

## 1. 项目范围
1. 用户端：注册、登录、个人信息、发帖、评论、点赞、图片上传。
2. 管理端：用户/帖子/评论治理（分页、详情、启用、禁用）。
3. 工程侧：统一鉴权、统一异常、统一返回协议、可回归测试。

## 2. 已实现能力
### 2.1 用户端
1. 账户模块完整可用（注册/登录/我的信息/修改资料）。
2. 帖子模块完整可用（增删改查 + 分页 + 公告分流）。
3. 评论模块完整可用（创建、分页、删除）。
4. 点赞模块完整可用（点赞、取消、去重）。
5. 文件模块完整可用（上传、绑定、解绑、定时清理）。

### 2.2 管理端
1. 管理员鉴权接口可用。
2. 用户治理接口可用。
3. 帖子治理接口可用。
4. 评论治理接口可用。

## 3. 核心业务规则
1. 管理员权限基于角色，不基于硬编码用户ID。
2. 业务删除默认采用软删除（状态切换）。
3. 评论回复关系使用双字段成对约束。
4. 点赞依赖数据库唯一约束防重复。
5. 文件状态机：`TEMP -> BOUND -> TEMP -> DELETED`。

## 4. 响应与异常约定
1. 统一返回三段式：`{ code, message, data }`。
2. 成功：`code=0`。
3. 失败：`code!=0`，并返回对应 HTTP 状态码。
4. 业务码到 HTTP 的映射由 `ErrorCodeToHttpStatus` 统一维护。

## 5. 验收结论
1. 后端功能满足课程交付要求。
2. 文档、接口、测试口径已统一。
3. 可进入前端联调与答辩演示阶段。
