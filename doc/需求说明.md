# Nuist CampusWall 需求说明（V2.1）

## 1. 项目背景
面向校园场景，提供“用户注册登录 + 帖子发布浏览 + 评论互动 + 点赞 + 管理治理”的内容平台后端。

## 2. 项目目标
1. 在课程交付周期内完成可演示、可联调、可答辩的后端系统。
2. 形成标准化文档体系（需求、架构、API、测试、数据字典、里程碑）。
3. 为后续 Vue 前端接入提供稳定接口。

## 3. 角色定义
### 3.1 普通用户
1. 注册与登录
2. 查看个人信息
3. 发布帖子、查看帖子
4. 发表评论、回复评论
5. 点赞/取消点赞（帖子、评论）

### 3.2 管理员
1. 登录管理端
2. 管理用户状态（启用/禁用）
3. 管理帖子状态（显示/隐藏）
4. 管理评论状态（显示/隐藏）

## 4. 当前实现状态（截至 2026-03-26）
### 4.1 已实现
1. 账户：注册、登录、`/me`
2. JWT：签发、解析、拦截鉴权
3. 帖子：创建、分页、详情
4. 评论：创建、分页
5. 点赞：点赞、取消点赞（支持帖子/评论）
6. 通用：统一返回体、全局异常、错误码常量化

### 4.2 未实现
1. 管理员治理接口
2. 参数校验深化（如 `@Email`）
3. 自动化测试体系
4. 前端页面联调

## 5. 功能需求
### 5.1 账户模块
1. 注册接口：用户名、教育邮箱去重
2. 登录接口：账号存在校验、状态校验、密码校验
3. 登录成功返回：`token + userInfo`
4. 当前用户接口：根据 JWT 返回用户信息

### 5.2 鉴权模块
1. 保护 `/api/**` 路径
2. 放行 `/api/account/register`、`/api/account/login`
3. 无 token 返回 `406`
4. token 无效/过期返回 `407`

### 5.3 帖子模块
1. 发布帖子（需登录）
2. 帖子分页（`pageNum/pageSize`）
3. 帖子详情（不存在返回 `409`）

### 5.4 评论模块
1. 发布评论（需登录）
2. 回复评论（`replyToCommentId/replyToUserId`）
3. 评论分页（按 `postId`）
4. 评论模型为“扁平楼层 + 回复字段”，不做无限树形

### 5.5 点赞模块
1. 点赞接口：`POST /api/like/do`
2. 取消点赞接口：`POST /api/like/undo`
3. 支持目标类型：`POST`、`COMMENT`
4. 幂等要求：重复点赞拦截，重复取消给出业务提示
5. 同步维护 `post.like_count` / `comment.like_count`

## 6. 数据模型约束
1. `comment.reply_to_comment_id` 与 `comment.reply_to_user_id` 必须“同时为空”或“同时非空”
2. `like` 表有唯一约束防重复点赞：`(user_id, target_type, target_id)`
3. 枚举字段持久化使用 `@EnumValue`

## 7. 非功能需求
1. 安全性：密码必须 BCrypt 存储，不允许明文
2. 可维护性：错误码统一常量管理
3. 可测试性：每个模块提供 HTTP 用例
4. 可扩展性：DTO 与 Domain 解耦

## 8. 验收标准
1. 注册、登录、`/me` 全链路可用
2. 帖子创建、分页、详情可用
3. 评论创建、分页可用
4. 点赞/取消点赞可用（帖子、评论）
5. 统一返回结构固定为 `code/message/data`
6. 文档与代码一致，无过期描述

## 9. 后续迭代重点
1. 管理员权限与后台治理
2. 点赞计数并发一致性优化（原子更新）
3. 前后端联调与回归测试
