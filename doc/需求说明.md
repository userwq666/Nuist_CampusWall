# Nuist CampusWall 需求说明（V2.0）

## 1. 项目背景
面向校园场景，提供“用户注册登录 + 帖子发布浏览 + 评论互动 + 点赞 + 管理治理”的内容平台后端。

## 2. 项目目标
1. 在课程交付周期内完成可演示、可联调、可答辩的后端系统。
2. 形成标准化文档体系（需求、架构、API、测试、数据字典、里程碑）。
3. 为后续 Vue 前端接入提供稳定接口。

## 3. 角色定义
### 3.1 普通用户
1. 注册与登录
2. 查看个人信息
3. 发布帖子
4. 查看帖子列表与详情
5. 发表评论、回复评论
6. 点赞/取消点赞（规划中）

### 3.2 管理员
1. 登录管理端
2. 管理用户状态（启用/禁用）
3. 管理帖子状态（显示/隐藏）
4. 管理评论状态（显示/隐藏）

## 4. 当前实现状态（截至 2026-03-25）
### 4.1 已实现
1. 账户：注册、登录、`/me`
2. JWT：签发、解析、拦截鉴权
3. 帖子：创建、分页、详情
4. 评论：创建、分页
5. 通用：统一返回体、全局异常、错误码常量化

### 4.2 未实现
1. 点赞模块（帖子点赞/评论点赞）
2. 管理员模块（治理接口）
3. 更细化的参数校验（如邮箱格式注解）
4. 前端页面与接口联调

## 5. 功能需求
### 5.1 账户模块
1. 注册接口：用户名、教育邮箱去重
2. 登录接口：账号存在校验、状态校验、密码校验
3. 登录成功返回：`token + userInfo`
4. 当前用户接口：根据 JWT 返回用户信息

### 5.2 鉴权模块
1. 保护 `/api/**` 路径
2. 放行 `/api/account/register`、`/api/account/login`
3. 无 token 返回 `406`
4. token 无效/过期返回 `407`

### 5.3 帖子模块
1. 发布帖子（需登录）
2. 帖子分页（`pageNum/pageSize`）
3. 帖子详情（不存在返回 `409`）

### 5.4 评论模块
1. 发布评论（需登录）
2. 回复评论（`replyToCommentId/replyToUserId`）
3. 评论分页（按 `postId`）
4. 评论模型为“扁平楼层 + 回复字段”，不做无限树形

### 5.5 点赞模块（规划）
1. 点赞行为记录在 `like` 表
2. 目标类型区分 `POST` / `COMMENT`
3. 使用唯一键 `(user_id, target_type, target_id)` 防重复点赞
4. `post.like_count`、`comment.like_count` 存冗余计数

## 6. 数据模型约束
1. `comment.reply_to_comment_id` 与 `comment.reply_to_user_id` 必须“同时为空”或“同时非空”
2. `like` 表有唯一约束防重复点赞
3. 枚举字段持久化使用 `@EnumValue`：
   - `Role`、`UserStatus`、`PostStatus`、`CommentStatus`、`TargetType`

## 7. 非功能需求
1. 安全性：密码必须 BCrypt 存储，不允许明文
2. 可维护性：错误码统一常量管理
3. 可测试性：每个模块提供 HTTP 用例
4. 可扩展性：DTO 与 Domain 解耦，便于前端字段演进

## 8. 验收标准
1. 注册、登录、`/me` 全链路可用
2. 帖子创建、分页、详情可用
3. 评论创建、分页可用
4. 统一返回结构固定为 `code/message/data`
5. 注册/登录/鉴权/参数异常均通过统一异常体系返回
6. 文档与代码一致，无“文档已写但代码未实现”项
