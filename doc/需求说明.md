# Nuist CampusWall 需求说明（V1.4）

## 1. 项目背景
为校园用户提供一个简洁的内容发布与互动平台，支持用户发帖、评论、回复与点赞，并为管理员提供基础内容治理能力。

## 2. 项目目标
1. 实现可演示、可交付的校园墙基础系统。
2. 覆盖核心业务链路：注册/登录 -> 发帖 -> 浏览 -> 评论/回复 -> 点赞。
3. 提供管理能力：用户管理、帖子管理、评论管理。

## 3. 当前实现进度
1. 已实现：注册接口（查重 + BCrypt + 入库）。
2. 已实现：登录接口（用户名查询 + 状态校验 + 密码校验）。
3. 已实现：登录返回 `LoginRespVO`（`token + userInfo`）。
4. 已实现：JWT 签发、解析与鉴权拦截。
5. 已实现：`GET /api/account/me` 当前用户接口。
6. 已实现：统一返回体与全局异常处理。
7. 已实现：错误码规范（401~408、422、500）。
8. 已实现：帖子发布接口（`POST /api/post/create`）。
9. 待实现：帖子列表/详情、评论、点赞完整接口。

## 4. 角色定义
### 4.1 普通用户
1. 注册账号并登录系统。
2. 发布帖子并浏览帖子列表/详情。
3. 发布评论并回复评论（扁平楼层模式）。
4. 对帖子或评论点赞/取消点赞。
5. 在个人中心查看个人信息与内容记录。

### 4.2 管理员
1. 登录管理后台。
2. 管理用户状态（启用/禁用）。
3. 管理帖子状态（显示/隐藏）。
4. 管理评论状态（显示/隐藏）。

## 5. 功能需求
### 5.1 用户模块
1. 注册：`POST /api/account/register`（已实现，错误码：401/402）。
2. 登录：`POST /api/account/login`（已实现，返回 `LoginRespVO`，错误码：403/404/405）。
3. 获取当前用户信息：`GET /api/account/me`（已实现，错误码：406/407/408）。

### 5.2 帖子模块
1. 发布帖子：`POST /api/post/create`（已实现，需 token，支持参数校验）。
2. 帖子列表（分页）。
3. 帖子详情。
4. 点赞/取消点赞（通过点赞记录表控制唯一性）。

### 5.3 评论模块
1. 发布评论。
2. 回复评论（`reply_to_comment_id` + `reply_to_user_id`）。
3. 评论列表（分页）。
4. 点赞/取消点赞。

### 5.4 管理模块
1. 用户状态管理。
2. 帖子状态管理。
3. 评论状态管理。

## 6. 数据模型说明（与当前脚本一致）
1. 用户表：`user`
2. 帖子表：`post`（含 `like_count`）
3. 评论表：`comment`（含 `like_count`、回复字段）
4. 点赞表：`like`（防重复点赞唯一约束）

评论回复一致性：
1. `reply_to_comment_id` 与 `reply_to_user_id` 必须同空或同非空。

点赞模型：
1. 记录行为：`like` 表（唯一键防重复）。

## 7. 非功能需求
1. 安全：密码 BCrypt 加密；后续接入 JWT。
2. 可维护性：统一返回结构、统一异常处理。
3. 可测试性：提供 HTTP 测试文件与可复现用例。

## 8. 验收标准
1. 注册接口可用，支持用户名与邮箱查重。
2. 登录接口可用，支持用户不存在/禁用/密码错误场景校验。
3. JWT 鉴权可用：登录返回 token，`/api/account/me` 可识别当前用户。
4. 发帖接口可用，未登录请求可被拦截，参数非法返回 `422`。
5. 密码不明文入库，登录通过 BCrypt 校验。
6. 接口成功返回统一结构（`code/message/data`）。
7. 业务异常通过全局异常处理统一返回。
8. 文档（README/需求/日志/数据字典/API/测试）与实现一致。
