# Nuist CampusWall 需求说明（V1.0）

## 1. 项目背景
为校园用户提供一个简洁的内容发布与浏览平台，支持用户发帖互动，并为管理员提供基础内容治理能力。
本项目采用前后端分离架构，后端以 Spring Boot 为核心实现业务服务。

## 2. 项目目标
1. 实现可演示、可交付的校园墙基础系统。
2. 覆盖核心业务链路：注册登录 -> 发帖 -> 浏览帖子 -> 个人中心。
3. 提供可用管理后台：用户管理、帖子管理。

## 3. 角色定义
### 3.1 普通用户
1. 注册账号并登录系统。
2. 发布帖子并查看帖子列表/详情。
3. 在个人中心查看个人信息与个人帖子。

### 3.2 管理员
1. 登录管理后台。
2. 查看并管理用户状态（启用/禁用）。
3. 查看并管理帖子状态（正常/下架/删除）。

## 4. 功能需求
### 4.1 用户模块
1. 用户注册
- 输入用户名、密码、昵称（可选头像）。
- 校验用户名唯一性。

2. 用户登录
- 使用用户名+密码登录。
- 登录成功后返回 Token（JWT）。

3. 用户信息
- 获取当前登录用户基本信息。
- 支持修改昵称、头像（先用 URL 方式）。

### 4.2 帖子模块
1. 发布帖子
- 字段：标题、正文、图片 URL（可选）。
- 仅登录用户可发布。

2. 帖子列表
- 分页查询。
- 按发布时间倒序展示。
- 仅展示状态为“正常”的帖子。

3. 帖子详情
- 展示标题、正文、作者、发布时间、图片。

### 4.3 个人中心模块
1. 我的信息
- 展示用户基础信息。

2. 我的帖子
- 分页查看当前用户发布的帖子。

3. 统计信息
- 展示当前用户发帖总数。

### 4.4 管理后台模块
1. 管理员登录
- 与普通用户登录分离或通过角色字段区分。

2. 用户管理
- 分页查看用户列表。
- 设置用户状态（启用/禁用）。

3. 帖子管理
- 分页查看帖子列表（含状态）。
- 下架帖子、删除帖子。

## 5. 非功能需求
1. 安全性
- 密码加密存储（BCrypt）。
- 接口鉴权采用 JWT。

2. 可维护性
- 统一接口返回格式。
- 统一异常处理。
- 分层架构清晰（controller/service/mapper/entity）。

3. 可演示性
- 提供 Swagger/Knife4j 接口文档。
- 关键功能可通过 Postman/Apifox 复现。

## 6. 数据范围（第一版）
1. 用户（user）
- 基础信息、角色、状态、创建时间。

2. 帖子（post）
- 标题、正文、作者、状态、发布时间。

3. 管理操作日志（admin_log，可选）
- 管理员操作类型、目标对象、时间。

## 7. 本期不纳入范围
1. 即时聊天与消息推送。
2. 复杂推荐算法。
3. 文件服务器上传（先使用图片 URL）。
4. 多角色细粒度权限（先实现用户/管理员两级）。

## 8. 验收标准
1. 用户可完成注册、登录并访问受保护接口。
2. 登录用户可发布帖子并在列表中查看。
3. 用户可在个人中心查看自己的帖子与统计信息。
4. 管理员可禁用用户、下架或删除帖子。
5. 系统具备可演示接口文档与基本异常处理机制。
