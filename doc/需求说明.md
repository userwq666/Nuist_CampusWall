# Nuist CampusWall 需求说明（V2.3）

## 1. 项目范围
1. 用户端：注册、登录、个人信息、发帖、评论、点赞、我的帖子/评论。
2. 管理端：用户管理、帖子管理、评论管理（下一阶段实现）。

## 2. 当前已实现需求
1. 注册：用户名/邮箱查重，密码加密存储。
2. 登录：密码校验成功后返回 `token + userInfo`。
3. 鉴权：登录后可访问受保护接口；未登录统一返回业务错误码。
4. 帖子：创建、分页、详情、修改、删除（软删除）。
5. 评论：创建、分页、删除（软删除）。
6. 点赞：点赞、取消点赞，支持帖子和评论。

## 3. 核心业务规则
1. 公开列表只展示 `status=ENABLE` 的帖子/评论。
2. 删除采用软删除（`status` 置为 `DISABLE`）。
3. 评论模型采用“扁平楼层 + 回复字段”：
   - `reply_to_comment_id`
   - `reply_to_user_id`
4. 回复一致性要求：两个字段要么同时为 `NULL`，要么同时非 `NULL`。
5. 点赞防重复：`like` 表唯一键 `(user_id, target_type, target_id)`。

## 4. 当前未实现需求
1. 管理员角色鉴权与 `/api/admin/**`。
2. 管理端列表筛选和状态流转接口。
3. 自动化测试（当前以 HTTP 回归为主）。

## 5. 验收标准（当前阶段）
1. 用户主链路可完整演示：注册 -> 登录 -> 发帖 -> 评论 -> 点赞。
2. 异常统一返回：`Result` + 业务 `code`。
3. 错误码与文档一致，HTTP 回归文件可复现。
4. 数据库结构与代码映射一致，基础索引齐全。
