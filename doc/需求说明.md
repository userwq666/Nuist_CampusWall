# Nuist CampusWall 需求说明（V3.3）

## 1. 项目范围
1. 用户端：注册、登录、当前用户、发帖、评论、点赞、文件上传、个人信息维护。
2. 管理端：管理员鉴权、用户治理、帖子治理、评论治理。
3. 工程侧：统一鉴权、统一异常、统一返回、可回归测试。

## 2. 已实现需求
### 2.1 用户端
1. 账号：`POST /api/account/register`、`POST /api/account/login`、`GET /api/account/my`、`POST /api/account/my/update`。
2. 文件：`POST /api/file/upload`（`fileType=POST/COMMENT/AVATAR`）。
3. 帖子：创建、分页、公告分页、我的分页、详情、修改、删除。
4. 评论：创建、分页、我的分页、删除。
5. 点赞：点赞、取消点赞。

### 2.2 管理端
1. 鉴权：`GET /api/admin/ping`。
2. 用户治理：分页、启用/禁用。
3. 帖子治理：分页、详情、启用/禁用。
4. 评论治理：分页、详情、启用/禁用。

## 3. 核心业务规则
1. `post/page` 返回普通帖子；`post/notice/page` 返回公告帖子。
2. 管理员权限基于角色 `Role.ADMIN` 判定。
3. 删除与治理采用状态切换（`ENABLE/DISABLE`）而非物理删除。
4. 点赞由联合唯一键防重复。
5. 评论回复字段 `reply_to_comment_id/reply_to_user_id` 成对约束。
6. 文件生命周期：`TEMP -> BOUND -> TEMP -> DELETED`。

## 4. 响应与异常规则
1. 统一返回体：`{ code, message, data }`。
2. 成功：`code=0`；失败：`code!=0`。
3. 失败时同时返回 HTTP 状态码（如 `401/403/404/409/422/500`）。
4. 错误映射由 `ErrorCodeToHttpStatus` 统一维护。

## 5. 非功能需求状态

2. 回归用例共 95 条，按模块覆盖核心链路。
