# Nuist CampusWall 数据字典（V3.4）

最后同步：2026-04-03
数据库：`campuswall`

## 1. user（用户表）
| 字段 | 类型 | 约束 | 默认值 | 说明 |
|---|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | - | 用户ID |
| username | VARCHAR(50) | NOT NULL, UNIQUE | - | 登录账号 |
| password | VARCHAR(100) | NOT NULL | - | BCrypt 密码 |
| nickname | VARCHAR(50) | NOT NULL | - | 昵称 |
| education_email | VARCHAR(100) | NOT NULL, UNIQUE | - | 教育邮箱 |
| image_url | VARCHAR(255) | NULL | NULL | 头像URL |
| role | TINYINT | NOT NULL | 0 | 角色：0用户，1管理员 |
| status | TINYINT | NOT NULL | 1 | 状态：1启用，0禁用 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

索引：
1. `idx_user_status_create_time(status, create_time)`
2. `idx_user_role_status(role, status)`

---

## 2. post（帖子表）
| 字段 | 类型 | 约束 | 默认值 | 说明 |
|---|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | - | 帖子ID |
| user_id | BIGINT | NOT NULL, FK->user.id | - | 作者用户ID |
| title | VARCHAR(100) | NOT NULL | - | 标题 |
| content | TEXT | NOT NULL | - | 正文 |
| image_url | VARCHAR(255) | NULL | NULL | 图片URL |
| status | TINYINT | NOT NULL | 1 | 状态：1显示，0隐藏 |
| like_count | INT | NOT NULL | 0 | 点赞数 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

索引：
1. `idx_post_user_id(user_id)`
2. `idx_post_status_create_time(status, create_time)`
3. `idx_post_user_status_create_time(user_id, status, create_time)`

---

## 3. comment（评论表）
| 字段 | 类型 | 约束 | 默认值 | 说明 |
|---|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | - | 评论ID |
| user_id | BIGINT | NOT NULL, FK->user.id | - | 评论用户ID |
| post_id | BIGINT | NOT NULL, FK->post.id | - | 所属帖子ID |
| reply_to_comment_id | BIGINT | NULL, FK->comment.id | NULL | 被回复评论ID |
| reply_to_user_id | BIGINT | NULL, FK->user.id | NULL | 被回复用户ID |
| content | VARCHAR(500) | NOT NULL | - | 评论内容 |
| image_url | VARCHAR(255) | NULL | NULL | 图片URL |
| status | TINYINT | NOT NULL | 1 | 状态：1显示，0隐藏 |
| like_count | INT | NOT NULL | 0 | 点赞数 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

索引：
1. `idx_comment_user_id(user_id)`
2. `idx_comment_post_id(post_id)`
3. `idx_comment_reply_to_comment_id(reply_to_comment_id)`
4. `idx_comment_reply_to_user_id(reply_to_user_id)`
5. `idx_comment_status_create_time(status, create_time)`
6. `idx_comment_post_status_create_time(post_id, status, create_time)`
7. `idx_comment_user_status_create_time(user_id, status, create_time)`

约束：
1. `ck_comment_reply_pair`：`reply_to_comment_id` 与 `reply_to_user_id` 成对为空或成对非空。

## 4. like（点赞表）
| 字段 | 类型 | 约束 | 默认值 | 说明 |
|---|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | - | 点赞记录ID |
| user_id | BIGINT | NOT NULL, FK->user.id | - | 点赞用户ID |
| target_type | TINYINT | NOT NULL | - | 目标类型：1帖子，2评论 |
| target_id | BIGINT | NOT NULL | - | 目标ID |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |

索引与约束：
1. `uk_user_target(user_id, target_type, target_id)`：防重复点赞。
2. `idx_target(target_type, target_id)`：按目标查询加速。
3. `idx_like_user_create_time(user_id, create_time)`：按用户时间查询加速。

---

## 5. file_asset（文件资源表）
| 字段 | 类型 | 约束 | 默认值 | 说明 |
|---|---|---|---|---|
| id | BIGINT | PK, AUTO_INCREMENT | - | 文件ID |
| owner_user_id | BIGINT | NOT NULL, FK->user.id | - | 上传用户ID |
| file_type | TINYINT | NOT NULL | - | 文件类型：1头像，2帖子图，3评论图 |
| biz_type | TINYINT | NULL | NULL | 业务类型：0空，1头像，2帖子，3评论 |
| biz_id | BIGINT | NULL | NULL | 业务ID |
| original_name | VARCHAR(255) | NOT NULL | - | 原始文件名 |
| object_key | VARCHAR(255) | NOT NULL, UNIQUE | - | 存储路径key |
| url | VARCHAR(500) | NOT NULL | - | 访问URL |
| mime_type | VARCHAR(100) | NOT NULL | - | MIME类型 |
| file_ext | VARCHAR(20) | NULL | NULL | 文件后缀 |
| file_size | BIGINT | NOT NULL | - | 文件大小(字节) |
| sha256 | CHAR(64) | NULL | NULL | 内容摘要 |
| status | TINYINT | NOT NULL | 0 | 状态：0临时，1已绑定，2已删除 |
| create_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NOT NULL | CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

索引：
1. `idx_file_asset_owner_status_create_time(owner_user_id, status, create_time)`
2. `idx_file_asset_file_type_status_create_time(file_type, status, create_time)`
3. `idx_file_asset_biz_type_biz_id(biz_type, biz_id)`
4. `idx_file_asset_sha256(sha256)`

约束：
1. `ck_file_asset_type`：`file_type in (1,2,3)`
2. `ck_file_asset_biz_type`：`biz_type is null or in (1,2,3)`（注：当前代码枚举含 `0=EMPTY`，若启用需同步SQL约束）
3. `ck_file_asset_status`：`status in (0,1,2)`
4. `ck_file_asset_bind_pair`：`biz_type/biz_id` 成对为空或成对非空

---

## 6. 枚举映射（代码真源）
1. `UserRole`：`0 -> USER`，`1 -> ADMIN`
2. `UserStatus/PostStatus/CommentStatus`：`1 -> ENABLE`，`0 -> DISABLE`
3. `LikeTargetType`：`1 -> POST`，`2 -> COMMENT`
4. `FileType`：`1 -> AVATAR`，`2 -> POST`，`3 -> COMMENT`
5. `FileBizType`：`0 -> EMPTY`，`1 -> AVATAR`，`2 -> POST`，`3 -> COMMENT`
6. `FileStatus`：`0 -> TEMP`，`1 -> BOUND`，`2 -> DELETED`

---
