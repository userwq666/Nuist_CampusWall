## 2026-03-19 ~ 2026-03-26｜内容主链路（帖子/评论/点赞）
### 目标
1. 完成校园墙核心业务功能。
2. 保证可分页、可维护、可回归。

### 本次实现
1. 帖子：创建、公开分页、我的分页、详情、修改、软删除。
2. 评论：创建、公开分页、我的分页、删除。
3. 点赞：点赞、取消点赞、重复操作校验。
4. 评论模型采用“扁平楼层 + 回复字段”方案。

### 关键设计决策
1. 删除策略采用软删除（状态位），不做物理删除。
2. 点赞去重由数据库唯一约束保障。
3. 查询主链路优先使用 MyBatis-Plus 条件构造。

### 验证
1. 内容模块主接口全部可跑通。
2. 关键异常场景（越权、重复操作、目标不存在）可稳定复现。

## 2026-04-03｜协议收口（HTTP + 业务码）
### 目标
1. 统一响应协议，便于前端长期维护。
2. 降低异常出口耦合。

### 本次实现
1. `Result<T>` 统一为三段式：`{ code, message, data }`。
2. 保留 HTTP 状态码语义（401/403/404/409/413/415/422/500 等）。
3. 新增映射类：`ErrorCodeToHttpStatus`（业务码 -> HTTP 状态）。
4. `GlobalExceptionHandler` 仅负责异常捕获与统一出参。
5. 前端 `request.js` 同步适配三段式返回。

### 回归与构建
1. 后端编译通过：`mvn -DskipTests compile`。
2. 前端构建通过：`npm run build`。
3. HTTP 用例口径统一为“HTTP 状态 + 业务 code 双断言”。
